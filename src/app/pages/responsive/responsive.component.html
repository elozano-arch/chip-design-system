Breakpoints

CHIP 2.0 utiliza cinco puntos de quiebre para adaptar la interfaz a distintos tamaños de pantalla. El enfoque es mobile-first: los estilos base aplican al tamaño más pequeño y se amplían progresivamente con min-width.

Grid de 12 columnas

El sistema de grilla se basa en 12 columnas flexibles con un gutter de 16px. En pantallas pequeñas, todas las columnas colapsan a 100% de ancho; en tablet se agrupan de a dos.

Buenas prácticas

Lineamientos para garantizar que las interfaces del sistema CHIP 2.0 funcionen correctamente en todos los dispositivos.

Mobile-first

Escriba los estilos base para pantallas pequeñas y use min-width para agregar complejidad en pantallas más grandes. Esto garantiza que la experiencia móvil sea la prioridad y no una adaptación posterior.

Unidades relativas (rem / em)

Prefiera rem y em sobre px para tipografía y espaciado. Esto permite que la interfaz escale correctamente cuando el usuario ajusta el tamaño de fuente del navegador.

min-width y max-width

Evite anchos fijos (width: 800px). Use max-width para limitar el crecimiento y min-width para evitar que los elementos se compriman demasiado. Combine con width: 100% para fluidez.

Touch targets: 44 × 44px mínimo

Todos los elementos interactivos (botones, enlaces, íconos clicables) deben tener un área táctil mínima de 44 × 44 píxeles según las pautas WCAG 2.5.5. Esto incluye padding suficiente aunque el contenido visible sea más pequeño.
